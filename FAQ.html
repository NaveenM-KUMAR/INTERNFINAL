Operating Systems & DBMS – Interview Q&A

Practice core OS and DBMS theory questions with short, interview-focused answers.

1. What is an Operating System (OS)?

An operating system is system software that manages computer hardware and software resources. It runs applications, controls memory, files, input/output devices, security, and provides user interfaces so users and programs can use the system.

2. Define functions of an OS.

Main functions include: process management, memory management, file system management, device/I-O management, security and protection, and providing user interfaces (CLI/GUI) for interaction.

3. What are the types of Operating Systems?

Common types: batch OS, multitasking/time-sharing OS, distributed OS, real-time OS, network OS, and mobile OS.

4. What is a process?

A process is a program in execution. It has its own address space, stack, registers, program counter, and is represented in the OS by a Process Control Block (PCB).

5. What is a thread?

A thread is a lightweight unit of execution within a process. Multiple threads in the same process share code and data sections but have their own stack and registers.

6. Explain process vs thread.

Processes have separate memory spaces and are heavier to create and switch. Threads share the same address space of a process, are lighter, and context switching between threads is faster. However, bugs in one thread can affect the whole process.

7. What is CPU scheduling?

CPU scheduling is the OS activity that selects which ready process should get the CPU next, aiming to optimise CPU utilisation, throughput, waiting time, and response time.

8. Name common CPU scheduling algorithms.

First-Come First-Served (FCFS), Shortest Job First (SJF), Shortest Remaining Time First (SRTF), Round Robin (RR), and Priority Scheduling (preemptive or non-preemptive).

9. Define memory management.

Memory management is the OS function of allocating and deallocating main memory to processes, keeping track of which parts of memory are in use, and preventing processes from interfering with each other’s memory.

10. What is paging?

Paging divides logical memory into fixed-size pages and physical memory into frames of the same size. Pages are mapped to frames via a page table, removing external fragmentation.

11. What is segmentation?

Segmentation divides memory into variable-sized logical segments such as code, stack, and heap. Each segment has a base and limit; external fragmentation can occur.

12. Define virtual memory.

Virtual memory is a memory management technique where the OS uses disk space as an extension of RAM, allowing large processes to execute even if only a part of them is in main memory at a time.

13. What is a critical section?

A critical section is a part of code where shared resources (like shared data) are accessed. Only one thread/process should execute its critical section at a time to avoid race conditions.

14. What is a semaphore?

A semaphore is a synchronisation primitive (integer variable) used to control access to shared resources. It uses atomic operations like wait(P) and signal(V) to implement mutual exclusion or resource counting.

15. What are the necessary conditions for deadlock?

Four Coffman conditions: mutual exclusion, hold and wait, no preemption, and circular wait. If all four hold simultaneously, deadlock can occur.

1. What is DBMS?

A Database Management System (DBMS) is software used to define, create, store, retrieve, and manage data in databases, while enforcing security, integrity, and efficient access.

2. What is RDBMS?

A Relational DBMS stores data in tables (relations) consisting of rows and columns and supports relationships between tables using keys. Examples include MySQL, Oracle, and PostgreSQL.

1. What is DBMS?

A Database Management System (DBMS) is software used to define, create, store, retrieve, and manage data in databases, while enforcing security, integrity, and efficient access.

2. What is RDBMS?

A Relational DBMS stores data in tables (relations) consisting of rows and columns and supports relationships between tables using keys. Examples include MySQL, Oracle, and PostgreSQL.

3. What are tables, tuples, and attributes?

A table is a collection of related data; a tuple is a row (record) in a table; an attribute is a column (field) describing a property of the entity.

4. Define Primary Key.

A primary key is a column or combination of columns that uniquely identifies each row in a table and cannot be NULL.

5. Explain Foreign Key.

A foreign key is a column in one table that refers to the primary key in another table. It creates a relationship between tables and enforces referential integrity.

6. What is normalization?

Normalization is the process of organizing data to reduce redundancy and update anomalies by splitting tables based on dependencies.

7. What are 1NF, 2NF, and 3NF?

1NF: each column holds atomic values, no repeating groups. 2NF: in 1NF and no partial dependency on a composite key. 3NF: in 2NF and no transitive dependency on non-key attributes.

8. Define ER diagram.

An Entity–Relationship (ER) diagram is a visual representation of entities, attributes, and relationships used for high-level database design.

9. What is ACID in database transactions?

ACID stands for Atomicity, Consistency, Isolation, and Durability – properties that ensure reliable, correct, and permanent database transactions.

10. Define a transaction.

A transaction is a logical unit of work that groups one or more database operations. It must either complete fully (COMMIT) or have all its changes undone (ROLLBACK).
